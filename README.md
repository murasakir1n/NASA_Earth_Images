# NASA EPIC Data Pipeline

## Описание проекта

В проекте реализован простой ETL-пайплайн, который получает метаданные снимков Земли из API NASA EPIC, обрабатывает их и сохраняет в базу данных PostgreSQL.

Пайплайн учитывает ограничения источника данных, такие как отсутствие данных за определённые даты.

---

## Архитектура

API → Extract → Transform → Load → PostgreSQL

---

## Основные возможности

* Получение данных из API NASA EPIC
* Обработка отсутствующих данных
* Преобразование вложенного JSON в реляционную структуру
* Защита от дубликатов (через ON CONFLICT)
* Простая и воспроизводимая структура пайплайна

---

## Особенности данных

API NASA EPIC не гарантирует наличие данных за каждую дату.

В проекте реализована логика:

* попытка загрузки данных за текущую дату
* при отсутствии данных — попытка загрузки за предыдущие дни
* при отсутствии данных загрузка пропускается без ошибок

---

## Структура базы данных

### Таблица images

* id — первичный ключ
* image_name — уникальное имя изображения
* caption — описание
* date — дата снимка

### Таблица position

* id — первичный ключ
* image_id — внешний ключ на images
* body — объект (sun, lunar, dscovr)
* x, y, z — координаты в системе J2000

---

## Запуск проекта

1. Создать файл `.env`:

DB=postgresql://user:password@localhost:5432/NASA_Earth_Images
API_KEY=your_api_key

2. Запустить:

python src/main.py
